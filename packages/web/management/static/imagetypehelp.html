Image Type Information

Single Partition

Single Partition partition images are support only a single NTFS partitions. These partitions are resizable, meaning that an image from a 200GB hard disk can be put on a 10GB hard disk as long as the data is less than 10GB. This image type is destructive, which means that your partitions will be modified, and it is possible if the task doesn't finish without error that your computer may not be bootable.

Overview
  • Recommended Method
  • Most Flexible
  • NTFS Only!
  • Destructive!
Multiple Partition - Single Disk

Multiple Partition - Single Disk will back up all the supported partitions on the first disk drive detected by FOG, but the partitions are NOT resizable by FOG. This means that the image must be restored to a disk of the same or larger capacity.

Overview
  • Single Disk Drive
  • Not Resizable
  • Supports: NTFS, XFS. JFS, HPFS, FAT16, FAT32, Reiser3, EXT2, EXT3
  • Linux Support!
Multiple Partition - All Disks

Multiple Partition - All Disks will back up all the supported partitions on all disk drives detected by FOG, but the partitions are NOT resizable by FOG. This means that the image must be restored to a disk of the same or larger capacity.

Overview
  • All Disk Drives
  • Not Resizable
  • Supports: NTFS, XFS. JFS, HPFS, FAT16, FAT32, Reiser3, EXT2, EXT3
  • Linux Support!
Raw Image

Raw Image will do a raw sector-by-sector copy of your hard disk, which means that if you have a 80GB hard drive, dd will process 80BG of data, and you will end up with a VERY LARGE image that is VERY SLOW to create and restore.

Overview
  • Single Drive
  • Not Resizable
  • Very Slow
  • Very Large Image
  • Works for nearly any FS
  • Maintains entire drive state, good for forensic evidence.
